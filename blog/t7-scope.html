JS Scope

In JavaScript, scope refers to the current context of your codes variables, functions and objects. Scopes can be globally or locally defined. When you first type a line of code in JavaScript you are in the Global Scope. A variable in the global scope can be accessed and used by any functions in the file. 

          
            // global scope
            var name = 'Nick';

If a variable is created inside of a function it is in theLocal Scope and is only defined inside that function, same with functions inside of functions. If you tried to call it outside of the function it would return undefined as the outside of that function can not see the variable. 

          
            var myFunction = function () {
              var name = 'Nick';
              console.log(name); // Nick
            };
            // Uncaught ReferenceError: name is not defined
            console.log(name);

Being in local scope could be compared to being inside of a car with tinted windows. No one outside of the car can see you or know it's you inside of the car, but you can see them and know what going on outside of the car. If you wanted you could pull over and use something from outside but no one could stop you and use anything inside your car (in a crime free world!).

Having good knowledge of local and global scope will allow you to understand where variables and functions are accessible, be able to change the scope of your codeâ€™s context and be able to write faster and more maintainable code, as well as debug much faster.
